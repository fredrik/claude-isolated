# Script Review Findings

## High

### Credentials source path mismatch
`scripts/claude-isolated:83` — Source path is `credentials.json` but the container expects `.credentials.json` (with leading dot). The local path should be `.credentials.json` to match.

### Hardcoded aarch64 in Containerfile
`Containerfile:15` — Zellij download URL is hardcoded to `aarch64-unknown-linux-musl`. Fails on x86_64 hosts. Should detect architecture via `TARGETARCH` build arg or runtime detection.

## Medium

### CLAUDE.md mounted but not validated
`scripts/claude-isolated:77` — `.claude/CLAUDE.md` is bind-mounted into the container but not included in the `required` list (line 55). If missing, podman will create it as a directory inside the container, breaking things. Make the mount conditional like credentials.

### cmd_stop swallows all errors
`scripts/claude-isolated:101-103` — Both `podman stop` and `podman rm` use `capture_output=True` with no return code check. Prints "Stopped and removed" even if the container doesn't exist.

## Low

### Unpinned install versions in Containerfile
`Containerfile:41-44` — Both `uv` and Claude Code are installed from latest. Builds are non-reproducible. Find the most recent version for each and pin it.

### Test uses hardcoded $HOME/tmp/
`tests/test-lifecycle.sh:23` — `mktemp -d "$HOME/tmp/..."` assumes `~/tmp/` exists. Using `/tmp/` or plain `mktemp -d` is more portable.

### Test doesn't exercise start subcommand
`tests/test-lifecycle.sh:30` — Container is started via `podman run` directly rather than through `cmd_start`, so the mount logic and name generation are untested.
